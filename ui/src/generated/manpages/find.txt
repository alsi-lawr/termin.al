NAME
     find - Find bounded virtual paths with direct predicates.

SYNOPSIS
     find [path] [-name pattern] [-path pattern] [-type f|d]
          [-maxdepth depth] [-mindepth depth]

DESCRIPTION
     Traverses  entries  beneath a starting directory and prints canonical paths
     that satisfy every predicate in command order. The default path is the cur‐
     rent pane directory. With no predicates,  every  traversed  entry  matches.
     Paths are resolved only inside the shared virtual filesystem.

OPTIONS
     -name pattern
            Match  each  entry's basename with *, ?, [abc], and ascending ranges
            such as [a-z].  * and ?  never cross  a  slash.  A  leading  dot  is
            matched  only  when its pattern segment begins with a dot. Malformed
            bracket constructs are literal; braces, extglob, recursive  **,  and
            regular expressions are not interpreted.

     -path pattern
            Match  the  canonical  ~ -rooted absolute virtual path with the same
            glob rules.

     -type f|d
            Match files, including locked files, or directories.

     -maxdepth depth
            Visit entries no deeper than depth.  The starting path has depth 0.

     -mindepth depth
            Exclude entries shallower than depth.  A value  of  1  excludes  the
            starting path.

OUTPUT
     The starting directory is included at depth 0 when it satisfies every pred‐
     icate. Descendants follow deterministic virtual-filesystem traversal order.
     No match produces no text output.

FAILURES
     Unsupported  options,  missing  predicate  values, invalid types or depths,
     more than one path, invalid or missing paths, files used  as  the  starting
     path,  and locked starting paths are rejected. Depths must be safe integers
     from 0 through 8.

LIMITS
     Traversal has a maximum depth of 8 and stops at  the  application's  recur‐
     sive-entry  limit,  currently 100 in the browser demo. Truncation keeps the
     matches already found and adds a runtime.truncated diagnostic.

NOTES
     Cancellation discards the command outcome. Locked descendants may be listed
     by path because find matches directory  entries  without  reading  document
     contents.

EXAMPLES
     $ find -name '*.md'
     $ find projects -type f -maxdepth 2
     $ find ~/notes -mindepth 1 -path '~/notes/*.md'
