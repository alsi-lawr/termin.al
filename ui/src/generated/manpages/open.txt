NAME
     open - Open virtual content inline or in a split viewer pane.

SYNOPSIS
     open [--split horizontal|vertical] <target>

DESCRIPTION
     Resolves  one target in the virtual filesystem and opens a viewer suited to
     its type. Files open as rendered Markdown, ordinary  directories  as  read-
     only  listings, and the special projects, blog and notes directories as hi‐
     erarchical collections.

     Without --split, the viewer temporarily replaces the current shell  presen‐
     tation.  A  split request creates an independent viewer pane and leaves the
     originating shell pane in place.

OPTIONS
     --split horizontal
            Create the viewer in  a  new  split  stacked  vertically  on  narrow
            screens and arranged horizontally at desktop width.

     --split vertical
            Create the viewer in a new vertical workspace split.

     The split option must be first and requires an orientation plus exactly one
     target. A target beginning with a hyphen may be passed after --.

VIEWER
     Rendered  documents  support  j/k or arrows, paging, g/G, / search, n/N and
     q/Escape return. Collections support movement,  collapse/expand,  filtering
     and Enter open. A directory viewer is a static listing closed with q or Es‐
     cape. Closing an inline viewer returns to its shell; closing a split viewer
     closes that pane, subject to the last-pane rule.

FAILURES
     Malformed  options,  invalid  or  missing targets, and locked files are re‐
     jected. Missing document content produces  the  stable  content-unavailable
     diagnostic. Cancellation opens nothing.

LIMITS
     Blog  and  notes  collection  targets traverse at most 100 entries to depth
     100. Other directory viewers list one directory level.

NOTES
     Targets are read-only virtual paths. Viewer navigation is transient.  Shell
     history  and  current directory remain owned by the originating shell pane,
     while a split viewer has no shell runtime.

EXAMPLES
     $ open about.md
     $ open --split vertical projects
     $ open ~/notes
